ETL MODERNISATION INITIATIVE — PROBLEM STATEMENT
Why EDF v6 Exists — The Problem and the Solution
🎯
One-Line Problem Statement
Transition from a code-driven Hadoop ETL ecosystem to a metadata-driven, governed and cloud-portable data platform that reduces engineering effort, standardizes pipelines, embeds governance and enables seamless migration to Databricks on AWS and GCP without rewriting existing pipelines.
7 DETAILED PROBLEM AREAS
🔴
High Dependency on Custom Code
Shell scripts, PySpark transformations and manual configuration patterns increase effort to onboard new pipelines. Reusable logic is limited. Standardization is low. Change management requires engineering intervention.
HIGH EFFORTNO REUSEMANUAL
🟠
Fragmented Configuration Management
Pipeline config resides in Oracle tables but transformation logic exists in PySpark code. Orchestration logic exists separately in Autosys JIL. This separation increases complexity in impact analysis and traceability.
FRAGMENTEDHARD TO TRACESPLIT TRUTH
🟡
Limited Native Governance Capabilities
Data lineage visibility is not automatically generated. Data profiling and quality validation are not standardized. Auditability across pipeline lifecycle is limited. Compliance readiness requires manual effort.
NO AUTO-LINEAGENO PROFILINGMANUAL AUDIT
🔵
Orchestration Overhead
Autosys job definitions must be manually created and maintained. Dependency management is effort intensive. Restartability and monitoring require manual intervention.
MANUAL JILNO AUTO-DEPSMANUAL RESTART
🟣
Scalability Constraints
Onboarding new data sources requires additional scripting. Transformation reuse is limited. Schema evolution handling is inconsistent. Operational effort increases linearly with pipeline count.
LINEAR EFFORTNO REUSEINCONSISTENT
🔴
Cloud Migration Risk
Existing pipelines are tightly coupled to Hive, Impala and on-prem Spark runtime. Migration to Databricks/AWS/GCP requires redesign of execution logic, storage format and orchestration. Lack of abstraction increases vendor lock-in risk.
TIGHTLY COUPLEDREDESIGN NEEDEDVENDOR LOCK-IN
🟠
Lack of Unified Framework
Different pipelines follow different patterns. No centralized metadata-driven control plane exists. No standardized pipeline lifecycle. Limited self-service capability for onboarding new datasets.
NO STANDARDSNO CONTROL PLANENO SELF-SERVICE
HOW EDF v6 SOLVES EACH PROBLEM
🗄Oracle as single config store
All pipeline behavior defined in Oracle tables. Zero BL in code. Adding a pipeline = INSERT rows. Changing logic = UPDATE a step_sql value.

One Python file handles every pipeline. Reads Oracle, executes steps, injects audit cols, writes encrypted Parquet. Zero pipeline-specific code.

✅Native DQ + profiling

⏰Dual scheduler auto-gen
Autosys JIL AND Airflow DAGs both generated from Oracle SCHEDULER_CONFIG. Dependency chains from DEPENDENCY_GRAPH. No manual JIL edits.
☁️2-statement cloud migration
ENVIRONMENT_CONFIG.active_flg = 2 SQL UPDATEs. TRANSFORM_STEPS SQL runs identically on any Spark platform. No pipeline redesign.
🚀Full migration toolkit
edf_migrate_audit.sh inventories existing jobs. PipelineMigrationConverter parses PySpark → suggests TRANSFORM_STEPS. jil_importer.py imports existing JIL.

  autosys_generator.py — reads Oracle SCHEDULER_CONFIG, emits Autosys JIL
class AutosysGenerator:
    def generate_all(self, namespace_id=None):
        pipelines = self.oracle.query("""
            SELECT p.pipeline_id, p.pipeline_nm,
                   s.start_time, s.autosys_box_nm,
                   s.machine, s.owner,
                   s.max_run_alarm, s.timezone, s.priority
            FROM   PIPELINE_REGISTRY p
            JOIN   SCHEDULER_CONFIG s ON s.pipeline_id=p.pipeline_id
            WHERE  p.active_flg='Y'
            AND    s.scheduler_type IN ('AUTOSYS','BOTH')
            AND    (:ns IS NULL OR p.namespace_id=:ns)
        """, ns=namespace_id)

def _make_jil(self, p, deps):
    condition = ""
    if deps:
        parts = [f"success({d.upstream_nm},+0)" for d in deps]
        condition = "condition:       " + " AND ".join(parts)

    return (
        f"insert_job: {p.pipeline_nm}   job_type: cmd\n"
        f"box_name:   {p.autosys_box_nm or 'EDF_DAILY_BOX'}\n"
        f"command:    /opt/edf/bin/edf_run.sh -p {p.pipeline_id}"
          f" -d %%date%% -m SCHEDULED\n"
        f"machine:    {p.machine or 'edf-spark-host'}\n"
        f"owner:      {p.owner}\n"
        f"permission: gx,ge,wx,we,mx,me\n"
        f"date_conditions: yes\n"
        f"days_of_week:    all\n"
        f"start_times:     \"{p.start_time or '06:00'}\"\n"
        f"{condition}\n"
        f"max_run_alarm:   {p.max_run_alarm or 120}\n"
        f"timezone:        {p.timezone or 'Europe/London'}\n"
        f"priority:        {p.priority or 50}\n"
        f"alarm_if_fail:   1\n"

  edf_migrate_convert.py — parses existing PySpark, suggests TRANSFORM_STEPS inserts
class PipelineMigrationConverter:
    """Semi-automated converter: parses existing PySpark code,
    suggests TRANSFORM_STEPS rows. Human reviews before INSERT."""

```
def analyse_spark_file(self, filepath, pipeline_id):
    with open(filepath) as f:
        source = f.read()
    tree = ast.parse(source)
    suggestions = []

    for node in ast.walk(tree):
        if isinstance(node, ast.Call):
            attr = getattr(node.func, 'attr', '')

            if attr == 'filter':        # FILTER step
                suggestions.append({
                    'step_type': 'FILTER',
                    'step_sql':  ast.unparse(node.args[0]),
                    'confidence': 'HIGH'
                })
            elif attr == 'withColumn':   # DERIVE step
                suggestions.append({
                    'step_type':     'DERIVE',
                    'step_sql':      ast.unparse(node.args[1]),
                    'target_alias':  ast.unparse(node.args[0]).strip("'"),
                    'confidence':    'HIGH'
                })
            elif attr == 'agg':          # AGGREGATE step
                suggestions.append({
                    'step_type':  'AGGREGATE',
                    'step_sql':   'REVIEW_REQUIRED',
                    'confidence': 'MEDIUM'
                })

    self._print_review_table(suggestions)
    return self.generate_insert_sql(pipeline_id, suggestions)

class JILImporter:
    """Parses existing Autosys JIL files and populates:
    - SCHEDULER_CONFIG (scheduling metadata)
    - DEPENDENCY_GRAPH (job conditions -> upstream deps)
    - PIPELINE_REGISTRY stubs for unregistered pipelines"""

```
def import_jil_file(self, jil_path):
    jobs = self._parse_jil(jil_path)
    for job in jobs:
        pipeline_id = self._get_or_create_pipeline(job)
        self._insert_scheduler_config(pipeline_id, job)
        for dep in job.get('conditions', []):
            upstream_id = self._get_pipeline_id(dep)
            if upstream_id:
                self._insert_dependency(upstream_id, pipeline_id)

def _parse_jil(self, path):
    jobs, cur = [], {}
    with open(path) as f:
        for line in f:
            line = line.strip()
            if line.startswith('insert_job:'):
                if cur: jobs.append(cur)
                cur = {'name': line.split()[1], 'conditions': []}
            elif line.startswith('start_times:'):
                cur['start_time'] = line.split('"')[1]
            elif line.startswith('condition:'):
                import re
                cur['conditions'] += re.findall(r'success\((\w+)', line)
            elif line.startswith('machine:'):
                cur['machine'] = line.split()[1]
            elif line.startswith('max_run_alarm:'):
                cur['max_run_alarm'] = int(line.split()[1])
    if cur: jobs.append(cur)
    return jobs

EDF v6 — MODULE · CIRCUIT BREAKER
Self-Healing Auto-Disable — 3 States, Config-Driven
RULE: After N consecutive failures (failure_threshold in CIRCUIT_BREAKER_CONFIG), breaker trips to OPEN — pipeline disabled automatically. After cooldown_minutes → HALF_OPEN (one probe run allowed). Success → CLOSED. Fail → back to OPEN. P1 alert + email on every state change.
🟢
CLOSED — Normal
Running normally. Failure count resets on success.
🔴
OPEN — Tripped · Disabled
N failures hit. Cooldown running. P1 alert fired.
🟡
HALF_OPEN — Probe
Cooldown expired. One probe run allowed.
  CIRCUIT_BREAKER_CONFIG + CIRCUIT_BREAKER_STATE DDL
CREATE TABLE CIRCUIT_BREAKER_CONFIG (

  failure_threshold NUMBER(3)   DEFAULT 3,   -- trips after N failures
  cooldown_minutes  NUMBER(5)   DEFAULT 30,  -- wait before HALF_OPEN
  probe_runs        NUMBER(2)   DEFAULT 1,   -- probes before CLOSED
  alert_on_trip     CHAR(1)     DEFAULT 'Y',
  alert_on_recover  CHAR(1)     DEFAULT 'Y',

);
